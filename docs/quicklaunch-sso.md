# QuickLaunch SSO Setup

This project supports signing in with QuickLaunch using the OIDC flow. The login
process is handled via `expo-auth-session` and exchanges the authorization code
for a Firebase custom token through your backend service.

## Steps

1. Obtain a QuickLaunch OIDC client for your institution.
2. Update `config.ts` with your QuickLaunch `client_id` and the authorization
   and token endpoints provided by QuickLaunch.
3. Set `QUICKLAUNCH_TOKEN_EXCHANGE_URL` to an endpoint on your backend that
   accepts the authorization `code` and returns a Firebase custom token.
4. Ensure the redirect URI generated by
   `AuthSession.makeRedirectUri({ useProxy: true })` is allowed in your
   QuickLaunch client configuration.
5. After configuration, users can tap **Login with QuickLaunch** on the login
   screen to authenticate with QuickLaunch.
